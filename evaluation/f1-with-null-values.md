# 결측값(null)이 낀 예측/정답을 F1에서 세는 법

구조화 추출 결과를 F1으로 채점할 때, 필드 값이 `null`(미추출·미확정)인 쌍을
어떻게 세느냐에 따라 점수 해석이 크게 달라진다.

## 두 가지 방식

| 방식 | null 낀 불일치 처리 | 무엇을 보여주나 |
|---|---|---|
| 표준 F1 | 아예 채점에서 제외 (모수에서 뺌) | "확실히 판단한 것만"의 정확도 |
| nan-inclusive F1 | 틀린 것으로 보고 **FP·FN 양쪽에 대칭 산입** | 미결정까지 대가로 친 상한 |

## 왜 대칭(FP+FN)인가
- 모델이 **있어야 할 값을 빼먹은 것**(예측=null, 정답=값) → FN
- 모델이 **엉뚱한 값을 넣은 것**(예측=값, 정답=null) → FP
- 한쪽만 세면 recall이나 precision 한쪽이 부풀려진다. 결측 불일치는
  "둘 다 벌주는" 쪽이 지표를 왜곡하지 않는다.

## 함정
- null 낀 쌍을 조용히 제외하면(표준 F1만 쓰면), **진짜 틀린 게 숨어서**
  점수가 실제보다 높게 나온다. "성능이 좋아졌다"의 상당 부분이 사실은
  "판단을 회피한 케이스가 늘었다"일 수 있다.
- 그래서 한 지표만 쓰지 말고 **표준 F1과 nan-inclusive F1을 나란히 병기**하는 게 안전하다.
  전자는 확신 있는 판단의 질, 후자는 미결정까지 포함한 하한을 본다.

## 덤: 채점 대칭성
- "사람 검토 대상(needs_review)"으로 유보된 예측은, 정답 쪽의 유보 데이터와
  **같은 규칙으로 채점에서 빼야** 부당하게 FP로 잡히지 않는다. 예측과 정답의
  유보 처리를 비대칭으로 두면 한쪽만 벌점을 먹는다.
